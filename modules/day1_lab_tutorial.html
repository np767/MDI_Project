Program and Package Installation

Before we can start creating our website with Quarto, we need to make sure we have the necessary programs and packages installed. Here are the steps to set up your environment: 1. Install R: If you haven’t already, download and install R from the CRAN website.

  1. Install RStudio: RStudio is an integrated development environment (IDE) for R. You can download it from the RStudio website.

  2. Install Python: If you plan to use Python for data processing or other tasks, you can download and install Python from the Python website. Make sure to add Python to your system PATH during installation.

  3. Install Jupyter Notebook: If you want to use Jupyter Notebooks for your project, you can install it using pip:

    pip install notebook

  4. Install Quarto: You can download and install Quarto from the Quarto website. Follow the installation instructions for your operating system.

  5. Install Required R Packages: Open RStudio and install the necessary packages for working with Quarto. You can use the following command to install the quarto package:

install.packages("quarto")
  1. Install Additional Packages: Depending on your project, you may need additional packages for data manipulation, visualization, or other tasks. You can install them using the install.packages() function in R. For example, if you need the ggplot2 package for data visualization

Environment Setup

Some of the things we installed in the previous section (like Python and R) are programming languages, while others (like Jupyter Notebook and Quarto) are programs that allow us to write and execute code in those languages. To keep our project organized and avoid conflicts between different projects, it’s a good idea to set up a virtual environment for our project.

A virtual environment (like venv for Python or renv for R) is a self-contained “sandbox” that contains all the necessary packages and dependencies for your project. Using a virtual environment helps to avoid conflicts between different projects and ensures that your project has access to the correct versions of packages.

Introduction to Directories

When creating a website with Quarto, it’s important to organize your files and directories properly. A common structure for a Quarto project might look like this:

├── index.qmd
├── about.qmd
├── contact.qmd
├── _quarto.yml
├── images/
│   ├── logo.png
│   └── banner.jpg
├── css/
│   └── styles.css
└── py/
    └── scripts.py

In this structure: - index.qmd, about.qmd, and contact.qmd are the main content files for your website. - _quarto.yml is the configuration file for your Quarto project. - The images directory contains any images you want to use on your website. - The css directory contains any custom stylesheets for your website. - The py directory contains any Python scripts you want to use for data processing or other tasks.

Working Directories

Your working directory is the folder where your current active files in your directory tree are. When you run commands in your terminal or execute code in your Quarto project, it will look for files relative to this working directory. It’s important to keep track of your working directory, especially when referencing files in your project.

To practice mapping paths, replace the example path below with the absolute path to your new project folder on your machine (e.g., C:/Users/YourName/... on Windows or /Users/YourName/... on Mac).

Code 
import os

# EXERCISE: Change the string below to match YOUR local computer path
os.chdir('/Users/nikhilpatla/Documents/DSAN/MDI/Week_1')

# Check the current working directory to confirm it worked
print("Current working directory:", os.getcwd())

Your working directory should be set to the root of your project, where your main .qmd files and _quarto.yml configuration file are located. This way, when you reference files in your project (like images or scripts), they will be found correctly.

This allows you to maintain a clean and organized project structure, making it easier to manage your website as it grows!

Setting up a Github Pages Repository

To create a website using Quarto and host it on GitHub Pages, you can follow these steps:

  1. Create a GitHub Repository: Go to GitHub and create a new repository. You can name it something like my-quarto-website. Make sure to initialize it with a README file.

  2. Clone the Repository: Clone the repository to your local machine using Git. You can use the command line or a Git client like GitHub Desktop.

    git clone https://github.com/your-username/my-quarto-website.git

  3. Set Up Quarto: Make sure you have Quarto installed on your machine. You can download it from the Quarto website.

  4. Create a Quarto Project: Navigate to the cloned repository and create a new Quarto project.

    cd my-quarto-website
quarto create-project --type website

  5. Add Content: Create your content using Quarto. You can create .qmd files for your pages. For example, you can create an index.qmd file for your homepage.

  6. Render the Website: Use the following command to render your Quarto project into a website.

    quarto render

  7. Commit and Push: After rendering, commit your changes and push them to GitHub.

    git add .
git commit -m "Update website"
git push origin main

  8. Enable GitHub Pages: Go to your repository on GitHub, navigate to the “Settings” tab, and scroll down to the “GitHub Pages” section. Select the branch you want to use (usually main) and the folder where your site is located (usually docs or the root). Save your changes.

  9. Access Your Website: After a few minutes, your website will be live at https://your-username.github.io/my-quarto-website/.

  10. Update Your Website: Whenever you want to update your website, simply make changes to your .qmd files, render the site again, and push the changes to GitHub.
